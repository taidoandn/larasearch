# Landing Screen Specification

This screen must follow the canonical design system in [../DESIGN.md](../DESIGN.md). The landing page may be slightly more brand-forward than product screens, but it must remain inside the same visual family. The parent document is the source of truth.

## Purpose

The landing page introduces Larasearch as a premium job-discovery product. It should bridge marketing and product: persuasive, but clearly grounded in the real search experience.

## Layout

- Use a top-to-bottom narrative structure:
  top bar, hero, capability strip, search preview, realistic query examples, technical proof section, final CTA.
- Keep each section visually connected to the same system used by the jobs product screens.
- Avoid introducing a second design language.

## Hero

- Lead with a concise headline and supporting line.
- The hero search entry must feel like a real product control, not a decorative marketing widget.
- Include compact operational metrics beneath the main search shell.

## Search Preview

- This section should preview the jobs index in a simplified, non-interactive form.
- Reuse the same visual logic as the jobs index:
  filter rail, toolbar metadata, soft result cards, skills pills, and match treatments.

## Query Examples

- Keep query examples realistic and product-oriented.
- Cards should stay compact and operational in tone.

## Technical Proof

- The technical section should feel like the same product in a darker, more technical mode.
- Use a code or execution-trace panel to signal search credibility and infrastructure maturity.

## Final CTA

- Keep the close simple and product-focused.
- The CTA should feel like the next step into the real search product, not a separate campaign action.
